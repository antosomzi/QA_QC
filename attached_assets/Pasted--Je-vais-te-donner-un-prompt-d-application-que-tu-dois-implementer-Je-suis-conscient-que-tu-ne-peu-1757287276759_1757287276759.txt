Spécifications fonctionnelles de l’application d’annotation

## Entrées

- Une **vidéo** (fichier local, ex. MP4).
- Un **fichier GPS associé** (CSV ou JSON) donnant la position (lat/lon) de chaque frame ou chaque timestamp.
- Optionnellement, un **fichier JSON d’annotations existantes** (pour modifier ou compléter).

## Interface utilisateur

- **Deux panneaux principaux** :
    - **Gauche → vidéo** : lecteur vidéo avec possibilité d’annoter en dessinant des bounding boxes (rectangles en pixels).
    - **Droite → carte** (Leaflet) : affiche les markers correspondant aux annotations.

## Fonctionnalités principales

- **Création d’annotation** :
    - L’utilisateur dessine une **bounding box** sur la vidéo.
    - Cette bbox est automatiquement associée à la **position GPS de la frame courante** (extraite du fichier GPS).
    - Un marker apparaît alors sur la carte.
- **Édition d’annotation** :
    - Déplacer/redimensionner la bbox directement dans la vidéo.
    - Déplacer le marker manuellement sur la carte (drag).
    - Cliquer sur un marker recentre la vidéo sur la **frame correspondante** et sélectionne la bbox associée.
    - Supprimer une annotation (bbox + marker).
- **Sauvegarde / chargement** :
    - Importer un JSON d’annotations existant.
    - Exporter toutes les annotations dans un JSON structuré.
    - Mode “sans vidéo” : possibilité d’ouvrir uniquement la carte avec les annotations déjà sauvegardées en base de données.

## Format de sortie JSON (proposition)

```json
{
  "video": {
    "video_id": "uuid-or-name",
    "fps": 30,
    "duration_ms": 123456
  },
  "annotations": [
    {
      "id": "ann-uuid-1",
      "frame_index": 42,
      "frame_timestamp_ms": 1730413526000,
      "gps": { "lat": 34.86281, "lon": -85.50273 },
      "bbox": {
        "x": 120, "y": 45, "width": 80, "height": 60, "unit": "pixel"
      },
      "label": "road_sign",
      "created_at": 1730413526000,
      "updated_at": 1730413527000
    }
  ]
}

```

# 📌 Stack et architecture

## Frontend

- **React + TypeScript**
- **Leaflet (react-leaflet)** pour la carte
- **Konva (react-konva)** pour dessiner et éditer les bounding boxes sur la vidéo
- **Video.js ou react-player** pour le lecteur vidéo
- **TailwindCSS** pour le styling rapide

## Backend

- **FastAPI (Python)**
- **PostgreSQL** pour stocker les vidéos et annotations
- **SQLAlchemy + GeoAlchemy2** pour l’ORM.
- **S3/MinIO** (ou stockage local) pour stocker les vidéos.
- API REST (et éventuellement WebSocket plus tard, mais pas nécessaire ici).

## Base de données

Deux tables principales :

- `videos` (id, metadata vidéo, chemin du fichier, fps, etc.)
- `annotations` (id, video_id, frame_index, timestamp, bbox en pixels, gps en POINT, métadonnées).

---

# 📌 Flux d’utilisation

1. L’utilisateur charge une **vidéo + fichier GPS** (et éventuellement un JSON d’annotations).
2. La vidéo est jouée dans le panneau gauche, le fichier GPS permet de récupérer la position pour chaque frame.
3. Lorsqu’une bbox est créée :
    - coordonnées bbox (pixels) + timestamp/frame_index → stockés,
    - GPS de la frame associée → utilisé pour le marker.
4. Le marker est ajouté sur la carte.
5. L’utilisateur peut ensuite **éditer bbox ou marker** librement.
6. À la fin, l’utilisateur exporte un **JSON complet** ou sauvegarde en **BD**.
